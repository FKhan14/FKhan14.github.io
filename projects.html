GitHub Talent Scout

Developer sourcing is notoriously inefficient — recruiters spend hours manually reviewing profiles with little signal on actual code quality. GitHub Talent Scout tackles this by building a talent matching system that analyzes real developer activity and code quality metrics rather than just keywords on a resume.

The system analyzes 927 GitHub profiles and draws insights from 289 real codebases using a custom Random Forest and Neural Network pipeline. On the backend, I built a full-stack search platform using PostgreSQL with pgvector, enabling sub-100ms semantic similarity search across 384-dimensional embeddings via a FastAPI backend.

Key Technologies

  • Random Forest & Neural Network (custom implementation)
  • PostgreSQL + pgvector for semantic search
  • FastAPI backend
  • 384-dimensional embeddings

Tech Salary Prediction Model

Salary transparency is a persistent problem in the tech industry — candidates often enter negotiations without reliable data on what their skills are worth. This project builds an end-to-end regression pipeline to predict tech salaries with high accuracy.

Using NumPy, Scikit-learn, and Pandas, I implemented a full data preprocessing and feature engineering pipeline that achieved an R² of 0.85 and a mean absolute error of approximately $8K — meaning predictions are typically within $8,000 of actual salaries.

Key Technologies

  • Python (NumPy, Pandas, Scikit-learn)
  • Feature engineering & preprocessing
  • Linear regression pipeline
  • R² = 0.85, MAE ≈ $8K

MedTap

Low-income clinics often lack access to fast, reliable medical reference tools. MedTap is an open-source medical retrieval-augmented generation (RAG) system designed to fill that gap, giving clinicians quick access to drug interaction data and medical literature.

The system integrates data from openFDA, PubMed, and DrugBank across 51 drugs and 328 validated drug interactions. I built the full pipeline from data collection through PubMedBERT embeddings and pgvector retrieval to a Streamlit web interface, achieving a Precision@15 of 0.69 on a 13-question held-out evaluation set.

Key Technologies

  • PubMedBERT embeddings + pgvector retrieval
  • openFDA, PubMed, DrugBank data sources
  • Streamlit web interface
  • RAG pipeline (end-to-end)
